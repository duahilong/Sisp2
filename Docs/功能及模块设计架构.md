# 项目各个模块以及功能架构

## 检查机制说明
- 项目中的检查能力不作为业务模块单独存在，而是作为基础能力，由主程序和各业务步骤按需调用。
- 检查机制仅用于执行前确认前置条件是否满足。
- 运行前检查由主程序在业务流程开始前统一执行。
- 若检查失败，则不进入对应的当前步骤执行流程。


## 运行前检查项目

- 管理员权限检查
- PowerShell 可用性检查
- 配置文件存在性检查
- 配置文件可解析性检查

## 运行前检查实现草案

- 入口位置：运行前检查由主程序统一调用，并在进入模块4、模块1、模块9等主流程步骤前执行。
- 职责范围：运行前检查仅负责确认程序是否具备启动和进入主流程的基本条件，不负责后续业务步骤的资源校验与执行结果验证。
- 执行顺序：
  1. 管理员权限检查
  2. PowerShell 可用性检查
  3. 配置文件存在性检查
  4. 配置文件可解析性检查
- 各检查项说明：
  - 管理员权限检查：确认当前程序是否以管理员权限运行；若未以管理员身份运行，则终止主流程。
  - PowerShell 可用性检查：确认当前环境可以成功调用 PowerShell，并能执行基础命令；若 PowerShell 不可用，则终止主流程。
  - 配置文件存在性检查：确认默认配置文件路径或主程序传入的配置文件路径存在；若文件不存在，则终止主流程。
  - 配置文件可解析性检查：在配置文件存在的前提下，复用模块4的配置读取能力验证配置文件是否可正常解析；若解析失败，则终止主流程。
- 失败策略：任意一项运行前检查失败，主程序均不进入后续业务流程，并直接输出失败原因。
- 输出形式建议：主程序在终端按顺序输出每项检查结果，建议包含“检查项名称、是否通过、失败原因”三类信息，最后再输出整体检查结论。

## 模块说明

#### 模块1：获取硬盘信息
- 功能：获取系统中所有硬盘的信息，包括硬盘类型、大小、盘符信息等。
- 输入：无
- 输出1：完整硬盘信息列表（供后续模块调用）
- 输出2：硬盘摘要信息列表（供终端显示和用户选择，包含：硬盘编号、硬盘型号、硬盘容量、分区表格式、连接方式、包含分区的盘符列表）

#### 模块2：初始化硬盘
- 功能：对硬盘进行初始化，检查硬盘分区表格式是否正确。
- 输入：硬盘信息列表
- 输出：初始化后的硬盘信息列表

#### 模块3：验证初始化结果
- 功能：验证初始化后的硬盘是否符合预期，包括检查分区表格式是否正确、是否符合预期的大小等。
- 输入：初始化后的硬盘信息列表
- 输出：验证结果（通过/失败）

#### 模块4：Json配置文件读取
- 功能：读取Json配置文件，获取分区信息，配置是否开启功能开关、镜像信息、调用外部软件的位置信息等。
- 输入：无 （传入配置文件路径）
- 输出：结构化配置数据（包含：配置文件路径、原始配置、分区信息、功能开关信息、镜像信息、外部软件路径信息、目录拷贝信息、排除硬盘名称列表）

#### 模块5：分区硬盘并格式化
- 功能：根据配置信息，对硬盘进行分区并格式化。
- 输入：配置信息 分区容量 分区格式 盘符信息
- 输出：分区后的硬盘信息列表

#### 模块6：镜像写入
- 功能：根据配置信息，调用ghost64.exe 对分区进行镜像写入。
- 输入：ghost64.exe路径 镜像路径 目标路径 盘符信息
- 输出：镜像写入成功与否（通过/失败）

#### 模块7：拷贝目录到特定位置
- 功能：根据配置信息，将指定目录下的所有文件拷贝到目标位置。
- 输入：硬盘编号 源目录 目标目录路径
- 输出：拷贝成功与否（通过/失败）

#### 模块8：创建引导记录
- 功能：根据配置信息，调用bcdboot.exe 创建引导记录。
- 输入：bcdboot.exe路径 目标路径 盘符信息
- 输出：引导记录创建成功与否（通过/失败）

#### 模块9：用户交互和硬盘信息显示
- 功能：与用户进行交互，包括提示用户选择硬盘、显示硬盘信息、选择硬盘、验证结果等。
- 输入：模块1返回的硬盘摘要信息列表；用户在终端输入硬盘选择指令，当前支持单个编号、范围输入（如 `1-3`）、多个编号输入（如 `1,3,5` 或 `1 3 5`）、`a` 表示选择全部磁盘、`q` 表示退出；支持中文逗号输入。
- 显示方式：终端使用无外框的对齐表格显示硬盘摘要信息；表头使用中文；默认使用 wcwidth 按终端显示宽度自动计算列宽，以保证中文表头与英文数据在终端中正确对齐。
- 输出：用户选择的硬盘编号（单个或多个）；若输入 `q` 则返回空选择结果。
