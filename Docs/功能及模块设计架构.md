# 项目各个模块以及功能架构

## 检查机制说明
- 项目中的检查能力不作为业务模块单独存在，而是作为公共基础能力，由主程序和各业务步骤按需调用。
- 检查机制分为两类：运行前检查、模块执行前校验。
- 运行前检查由主程序在业务流程开始前统一执行，用于检查管理员权限、PowerShell 可用性、配置文件是否存在、外部软件路径是否有效等基础条件。
- 模块执行前校验由具体步骤在执行前按需调用，用于检查该步骤所需条件是否满足，例如镜像写入前检查 `ghost64.exe` 和镜像文件是否存在、创建引导前检查 `bcdboot.exe` 和目标路径是否有效等。
- 若检查失败，则不进入对应的后续执行流程。
- 随着项目功能扩展，新的检查需求应继续纳入现有检查机制中，并根据触发时机归类为“运行前检查”或“模块执行前校验”。

## 当前检查项目清单

### 运行前检查项目
- 管理员权限检查。
- PowerShell 可用性检查。
- JSON 配置文件是否存在。
- JSON 配置文件是否可以正确解析。
- `ghost64.exe` 路径配置是否有效。
- `bcdboot.exe` 路径配置是否有效。

### 模块执行前校验项目
#### 模块1：获取硬盘信息
- 检查 PowerShell 磁盘相关命令是否可正常调用。

#### 模块5：分区硬盘并格式化
- 检查目标硬盘是否已由用户正确选择。
- 检查分区配置是否完整且格式合法。

#### 模块6：镜像写入
- 检查 `ghost64.exe` 文件是否存在。
- 检查镜像文件是否存在。
- 检查目标分区或目标盘符是否存在且可用。

#### 模块7：拷贝目录到特定位置
- 检查源目录是否存在。
- 检查目标路径是否有效且可写。

#### 模块8：创建引导记录
- 检查 `bcdboot.exe` 文件是否存在。
- 检查目标路径是否有效。
- 检查 EFI 分区或引导目标分区是否存在且可用。

## 模块说明

#### 模块1：获取硬盘信息
- 功能：获取系统中所有硬盘的信息，包括硬盘类型、大小、盘符信息等。
- 输入：无
- 输出1：完整硬盘信息列表（供后续模块调用）
- 输出2：硬盘摘要信息列表（供终端显示和用户选择，包含：硬盘编号、硬盘型号、硬盘容量、分区表格式、连接方式、包含分区的盘符列表）

#### 模块2：初始化硬盘
- 功能：对硬盘进行初始化，检查硬盘分区表格式是否正确。
- 输入：硬盘信息列表
- 输出：初始化后的硬盘信息列表

#### 模块3：验证初始化结果
- 功能：验证初始化后的硬盘是否符合预期，包括检查分区表格式是否正确、是否符合预期的大小等。
- 输入：初始化后的硬盘信息列表
- 输出：验证结果（通过/失败）

#### 模块4：Json配置文件读取
- 功能：读取Json配置文件，获取分区信息，配置是否开启功能开关、镜像信息、调用外部软件的位置信息等。
- 输入：无
- 输出：配置信息 分区信息 功能开关信息 镜像信息 调用外部软件的位置信息等

#### 模块5：分区硬盘并格式化
- 功能：根据配置信息，对硬盘进行分区并格式化。
- 输入：配置信息 分区容量 分区格式 盘符信息
- 输出：分区后的硬盘信息列表

#### 模块6：镜像写入
- 功能：根据配置信息，调用ghost64.exe 对分区进行镜像写入。
- 输入：ghost64.exe路径 镜像路径 目标路径 盘符信息
- 输出：镜像写入成功与否（通过/失败）

#### 模块7：拷贝目录到特定位置
- 功能：根据配置信息，将指定目录下的所有文件拷贝到目标位置。
- 输入：硬盘编号 源目录 目标目录路径
- 输出：拷贝成功与否（通过/失败）

#### 模块8：创建引导记录
- 功能：根据配置信息，调用bcdboot.exe 创建引导记录。
- 输入：bcdboot.exe路径 目标路径 盘符信息
- 输出：引导记录创建成功与否（通过/失败）

#### 模块9：用户交互和硬盘信息显示
- 功能：与用户进行交互，包括提示用户选择硬盘、显示硬盘信息、选择硬盘、验证结果等。
- 输入：模块1返回的硬盘摘要信息列表
- 输出：用户选择的硬盘编号（单个或多个）
