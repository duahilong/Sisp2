# 项目名称 ：Sisp - System Installation Script Project

## 整体介绍
结合编写的工具、脚本和外部软件，来实现对多个或者单个硬盘的系统安装。通过对硬盘的初始化、格式化、分区、克隆操作系统等一系列操作，实现对硬盘的完整制作。


## 项目技术架构
项目整体使用 Python 编写，结合外部软件和 Windows PowerShell 命令，实现完整流程。
- 工具：ghost64.exe、bcdboot.exe 等，更详细的说明 [引用外部软件信息.md](引用外部软件信息.md)
- 脚本：硬盘分区相关操作通过 Windows PowerShell 命令执行。
- 模块：通过 Python 编写模块，来实现各个功能。

## 运行前置条件
- 项目运行环境为 Windows。
- 项目依赖 PowerShell 执行磁盘相关命令。
- 程序需以管理员权限运行。
- 若未以管理员权限运行，程序应在流程开始前终止，并提示用户重新以管理员身份启动。

## Python 项目依赖
- 当前项目使用 `requirements.txt` 管理 Python 依赖。
- 当前已声明依赖：`wcwidth>=0.2.13`
- 初次运行项目前，建议先执行：`py -m pip install -r requirements.txt`

## 运行方式与 JSON 配置文件
- 主程序默认读取 `json/win11.json` 配置文件。
- 可直接执行默认方式运行：`py .\app\main.py`
- 若需要指定其他 JSON 配置文件，可通过 `-j` 参数传入路径。
- 示例：`py .\app\main.py -j .\json\win11.json`
- 示例：`py .\app\main.py -j D:\Code-Project\Sisp2\json\win11.json`
- 多硬盘执行时，主程序会为每个被选择的硬盘启动独立 PowerShell worker 窗口。
- worker 模式主要由主程序自动启动，也可用于单盘调试：`py .\app\main.py --worker-disk 2 -j .\json\win11.json`。

## 磁盘操作安全规则

- 默认禁止操作系统盘、启动盘和配置中排除的硬盘。
- 当前策略是用户选择可选硬盘后直接进入后续模块。
- 破坏性操作前必须显示目标硬盘编号、型号、容量、盘符等关键信息。
- 任一步失败后必须中止后续破坏性操作，并输出失败原因。
- 初始化和分区步骤会在 PowerShell 层再次校验目标硬盘不是系统盘、启动盘、离线盘或只读盘。

## 多硬盘 worker 执行说明

- 用户只选择一个硬盘时，主程序在当前窗口直接执行后续流程。
- 用户选择多个硬盘时，主程序不在当前窗口直接执行磁盘操作，而是为每个硬盘启动一个独立 PowerShell worker 窗口。
- 每个 worker 窗口只处理一个硬盘，按完整流程执行当前已实现阶段。
- 每个 worker 会重新执行运行前检查、重新读取配置、重新扫描硬盘并重新应用磁盘保护。
- 主程序自动启动 worker 时，会传入目标硬盘的 `UniqueId`、`SerialNumber`、型号和容量；worker 重新扫描后会进行硬盘身份二次确认，防止磁盘编号变化导致误操作。
- 主程序启动多个 worker 时，会在相邻 worker 之间等待 3 秒，降低多个 USB 硬盘同时执行初始化和分区时的存储服务抖动。
- worker 之间互不共享执行状态，便于分别查看每个硬盘的执行结果。
- worker 窗口执行完成后默认保持打开，方便查看输出和错误信息。

## 测试安全说明

- 默认安全测试入口为：`py .\tests\run_all.py`。
- 默认安全测试使用模拟磁盘数据，不读取或修改本机真实硬盘。
- `tests/test_disk_info_raw.py` 会读取本机真实磁盘信息，仅作为手动诊断工具使用，不包含在默认安全测试入口中。
- 默认安全测试会覆盖初始化、分区和验证逻辑，但通过 mock 和脚本文本断言完成，不会对真实硬盘执行 `Clear-Disk`、`Initialize-Disk`、`New-Partition` 或 `Format-Volume`。
